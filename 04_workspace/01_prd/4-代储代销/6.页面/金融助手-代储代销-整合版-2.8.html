说明：
业务全流程：提交申请 → 待审核 → 审核通过 → 入库验收 →仓单生效（普通存单）→ 保证金拨付 → 定向销售（通过市场助手）→ 出库发货 → 确认收款
核心流程：提交申请 → 待审核 → 审核通过 → 生成批次编号 → 根据SKU拆单生成代储订单编码 → 入库验收 → 仓单生效（普通存单）→ 保证金拨付 → 定向销售（通过市场助手）→ 出库发货 → 确认收款
分支流程：待审核 → 已拒绝（流程结束，不可再次申请同批次货物）；待审核 → 已驳回（供应商修改后回到待审核）
关键节点：审核状态定义、编号规则、接口调用、权限控制
1. 状态定义：
     待审核：供应商提交申请，等待审核人员审核
     已通过：审核人员审核通过，生成批次编号
     已拒绝：审核人员审核拒绝，流程结束，不可再次申请同批次货物
     已驳回：审核人员要求供应商补充材料，修改后回到待审核
2. 状态流转：
     待审核 → 已通过 / 已拒绝 / 已驳回
    已驳回 → 待审核（供应商修改后）
3. 编号规则：
    申请编号：SQ + YYYYMMDD + 4位流水号（当天唯一）
    批次编号：DCPC + YYYYMMDD + 4位流水号（当天唯一）
    代储订单编码：DC + YYYYMMDD + 4位流水号（当天唯一）
    批次编号在审核完成后生成，代储订单编码在审核完成后根据SKU拆单生成；
    关系如下：一个申请编号对应一个批次编号，一个批次编号对应多个代储订单编码（根据SKU拆单）
4. 接口调用：审核通过后推送消息至仓配助手（接口文档见:代提供）
5. 权限控制：待审核状态显示"审核"按钮，全状态显示"详情"

说明：
1. 仓单状态说明：
    • 待生效：仓单已创建但尚未生效，可能正在审核中。审批通过后生成代储订单号，同步生成仓单记录，状态为待生效。
    • 普通存单：仓单已生效，处于正常状态，未被质押或释放。入库完成后，仓单状态变为普通存单，表示货物已安全存放在仓库中。
    • 已质押：仓单已被质押，货物处于质押状态，无法进行释放操作。保证金拨付后，仓单状态变为已质押，表示货物已被质押给金融机构，等待后续的保证金拨付和定向销售流程。
    • 部分释放：仓单部分货物已被释放，剩余部分仍处于锁定状态。部分销售后，仓单状态变为部分释放，表示部分货物已被销售出库，剩余货物仍在仓库中等待后续处理。
    • 已核销：仓单已完成核销，货物已出库或订单已完成，仓单不再具有流通性。全部销售完成后，仓单状态变为已核销，表示货物已全部销售出库，订单已完成，仓单不再具有流通性。
2. 状态流转
    • 仓单初始状态为"待生效"，货物入库后变为"普通存单"。
    • 在"普通存单"状态下，仓单可以被质押，质押后状态变为"已质押"。
    • 已质押的仓单可以被部分释放，释放后状态变为"部分释放"。
    • 仓单完成核销后状态变为"已核销"。
    • 待生效 → 普通存单 → 已质押 → 部分释放 → 已核销
3. 编码规则
    • 仓单编号（CD+年月日+4位流水号，例如：CD20250909001）
    • 入库清单编号（RK+年月日+4位流水号，例如：RK20250909001）
4. 接口，仓配同步入库信息
    • 仓配系统通过接口将入库信息同步到金融助手系统，金融助手系统根据入库信息补充对应的仓单记录。

说明：
1. 保证金状态说明
    待拨付：审批通过并且货物已经入库完成，接收到入库信息，等待财务拨付保证金。
    已拨付：财务已完成拨付操作，保证金已进入供应商平安钱包账户。
2. 状态流转说明
    待拨付 → 已拨付：当财务完成拨付操作后，系统自动更新状态为"已拨付"。
3. 编码规则
    拨付编号（Pay No）：以"BF"开头，后跟8位日期（YYYYMMDD）和3位流水号，例如：BF20250909001。
4. 字段说明
    实际入库货值：系统根据入库单信息计算的货物实际入库价值。计算公式为：实际入库货值 = 入库数量 × 评估单价。
    保证金比例：根据平台规则设定的保证金比例，例如：70%。取值审核中的保证金比例。
    拨付金额：根据实际入库货值和保证金比例计算得出的保证金拨付金额，计算公式为：拨付金额 = 实际入库货值 × 保证金比例。
5. 拨付流程
    财务人员在系统中查看待拨付的保证金信息，点击"拨付"操作后，系统弹出拨付确认弹窗，显示拨付编号、供应商名称、拨付金额和收款账户信息等重要信息。
    财务人员确认信息无误后，点击"确认拨付"按钮，系统执行拨付操作，并自动更新保证金状态为"已拨付"，同时记录拨付时间和操作人信息。
6. 拨付异常说明
    如果拨付过程中出现异常情况，例如银行接口调用失败、账户信息错误等，系统会提示相应的错误信息，并允许财务人员重新尝试拨付操作。
7. 拨付数据生成说明
    保证金拨付数据由系统根据入库单信息和平台规则自动生成。当仓单状态更新为"普通存单"时，系统自动计算实际入库货值和拨付金额，并生成对应的保证金拨付记录，初始状态为"待拨付"。
    一条代储订单对应一条保证金拨付记录，保证金拨付记录与代储订单通过代储订单编码进行关联。系统根据入库单信息自动更新保证金拨付记录中的实际入库货值和拨付金额等字段信息。

代储订单销售规则
1.销售前提：订单拨付状态 = 已拨付
2.销售渠道：市场助手（一期），分销平台/购机通（二期）
3.销售单价限制：
    -单台服务费=评估单价*保证金比例*服务费率*（下单付款日期（当前日期）-保证金拨付日期+1）
    -单台仓储费=仓储费单价*（下单付款日期（当前日期）-保证金拨付日期+1）
    -单台最低销售单价=max（评估单价*保证金比例*（1+0.2%),评估单价*保证金比例+单台服务费+单台仓储费）；保留两位小数，向上取
    - 销售单价不得低于最低销售单价，前端校验+后端拦截
4.拨付划扣：
    - 本次划扣保证金：
      1. 未全部销售完：本次划扣保证金=评估单价*保证金比例*销售数量
      2. 全部销售完：本次划扣保证金=拨付保证金总额-已收保证金（已付款销售单+未付款销售单）；注：前面有订单没有付款完成前，不能在发起销售单
    -本次仓储费=单台仓储费*销售数量
    -本次服务费=单台最低销售价*销售数量-本次划扣保证金-本次仓储费
    -剩余金额=实际销售单价*销售数量-单台最低销售单价*销售数量
5.销售数量限制：销售数量 ≤ 库存数量
6.接口调用：确认下单后 调用接口，同步至市场助手
